Smart Campus IoT Log Analytics (From MapReduce to Ray)
Project Overview

This mini-project demonstrates a simplified cloud-based IoT analytics workflow using cloud object storage, MapReduce-style batch analytics, and Ray-based parallel processing.

The project analyses a synthetic smart-campus IoT log dataset generated by campus IoT devices located in different buildings and rooms.

The workflow of the project is:

IoT Dataset
    ↓
Amazon S3 Object Storage
    ↓
MapReduce Baseline Analytics
    ↓
Ray Parallel Analytics
    ↓
Result Comparison

The project focuses on:

cloud object storage,
batch analytics using MapReduce concepts,
parallel processing using Ray,
and comparison between traditional and modern distributed analytics frameworks.
Dataset

The dataset file used in this project is:

Comp3006J MiniProject 2 Dataset.csv

The dataset contains synthetic smart-campus IoT logs.

Each record contains:

timestamp
device_id
building
floor
room
sensor_type
event_type
value
status
battery_level

Example record:

2026-04-10T09:15:22Z,D023,Library,2,R204,temperature,reading,27.8,OK,74
Cloud Object Storage

The dataset was uploaded to Amazon S3 before analytics processing.

Amazon S3 was selected because it provides:

scalable cloud storage,
high durability and availability,
efficient storage for large datasets,
and integration with cloud-based analytics workflows.

A local copy of the dataset was later used for analytics execution.

Installation

Install dependencies:

pip install -r requirements.txt

If Ray is not installed:

pip install ray
Running the Project

Open terminal in the project root directory:

cd smart-campus-iot-analytics
MapReduce Analytics
1. Event Count by Sensor Type

Run:

python mapreduce/sensor_count.py

Output:

outputs/sensor_count.txt

This task counts how many records belong to each sensor type.

2. Warning/Error Count by Building

Run:

python mapreduce/building_warning_error.py

Output:

outputs/building_warning_error.txt

This task counts WARNING and ERROR records grouped by building.

3. Top 10 Most Active Devices

Run:

python mapreduce/top_devices.py

Output:

outputs/top_devices.txt

This task identifies the devices that generated the largest number of log records.

Ray Extension Analytics
Abnormal Device Detection

Run:

python ray_extension/abnormal_detection_ray.py

Output:

outputs/abnormal_devices.csv

A device is considered abnormal if at least one of the following conditions is satisfied:

battery_level < 20
ERROR status appears at least 3 times
temperature value > 32 appears at least 3 times

The Ray implementation uses:

@ray.remote

functions for parallel task execution.

The dataset is divided into chunks and processed in parallel before combining the final abnormal-device results.
